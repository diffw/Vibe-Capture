# Vibe Capture 产品规格说明书

---

## 一、产品定位

### 1.1 产品简介

**Vibe Capture** 是一款面向开发者和设计师的 macOS 效率工具，专注于**截图 + Prompt 一键发送**场景，帮助用户快速将屏幕截图与文字描述发送到 AI 编程助手、设计工具和通讯应用中。

### 1.2 目标用户

| 用户群体 | 典型场景 |
|---------|---------|
| **AI 辅助编程开发者** | 截图 UI Bug → 发送到 Cursor/VS Code 让 AI 修复 |
| **设计师** | 截图设计稿 → 发送到 Figma Make 生成代码 |
| **产品/运营** | 截图问题 → 发送到 Claude 分析或 Telegram 沟通 |

### 1.3 产品价值主张

> **"截图即 Prompt"** —— 将传统的「截图 → 保存 → 打开应用 → 粘贴 → 输入描述」5 步流程压缩为 **1 步**

**核心差异化**：
- 图片 + 文字**一次性粘贴**到目标应用输入框
- 智能识别**上一个活跃应用**作为默认目标
- 支持**多应用切换**，无需重复截图

## 二、产品功能

### 2.1 核心功能架构

```
┌─────────────────────────────────────────────────────┐
│                   Vibe Capture                       │
├─────────────────────────────────────────────────────┤
│  ┌───────────┐   ┌───────────┐   ┌───────────┐     │
│  │  截图引擎  │ → │ Prompt 编辑│ → │ 智能粘贴  │     │
│  └───────────┘   └───────────┘   └───────────┘     │
│        ↓               ↓               ↓            │
│   区域选择        文字输入         目标应用         │
│   全局快捷键      快捷发送         自动激活         │
│                                   模拟 ⌘V          │
└─────────────────────────────────────────────────────┘
```

### 2.2 功能清单

#### 2.2.1 截图功能

| 功能 | 说明 |
|-----|------|
| **全局快捷键** | 默认 `⌘ + Shift + 1`，可自定义 |
| **区域选择** | 鼠标拖拽选择任意屏幕区域 |
| **多显示器支持** | 支持在任意显示器上截图 |
| **ESC 取消** | 按 ESC 键取消当前截图 |

#### 2.2.2 Prompt 编辑

| 功能 | 说明 |
|-----|------|
| **文字输入框** | 多行输入，支持换行 |
| **占位提示** | "Describe what should be changed. [App] will see both the image and this text." |
| **快捷发送** | `⌘ + Enter` 快速发送 |

#### 2.2.3 目标应用

| 功能 | 说明 |
|-----|------|
| **智能识别** | 自动识别截图前的活跃应用作为默认目标 |
| **应用切换** | 下拉菜单选择其他目标应用 |
| **白名单机制** | 仅支持预设的应用列表 |

**支持的应用列表**：

| 类型 | 应用 |
|-----|------|
| AI 编程 | Cursor, VS Code |
| AI 对话 | Claude (Desktop) |
| 设计工具 | Figma (含 Figma Make) |
| 即时通讯 | Telegram |
| 浏览器 | Chrome, Safari, Edge, Arc |

#### 2.2.4 保存功能

| 功能 | 说明 |
|-----|------|
| **手动保存** | 点击 Save 按钮保存截图 |
| **自定义路径** | 可在设置中指定默认保存文件夹 |
| **文件命名** | `screenshot_YYYYMMDD_HHmmss.png` |

#### 2.2.5 系统设置

| 设置项 | 说明 |
|--------|------|
| **全局快捷键** | 自定义触发截图的快捷键 |
| **开机自启** | 登录时自动启动 |
| **保存路径** | 设置截图默认保存位置 |

## 四、用户流程

### 4.1 主流程

```
用户在 Cursor 中编码
        ↓
发现 UI 问题，需要 AI 帮助修复
        ↓
按下 ⌘ + Shift + 1
        ↓
拖拽选择问题区域
        ↓
弹出编辑窗口，显示截图预览
        ↓
输入 Prompt: "这个按钮的样式不对，请修复"
        ↓
按 ⌘ + Enter 或点击 [Paste to Cursor]
        ↓
自动切换到 Cursor，图片+文字粘贴到输入框
        ↓
用户在 Cursor 中按 Enter 发送给 AI
```

### 4.2 切换目标应用

```
截图完成，默认目标是 Cursor
        ↓
用户想发送到 Claude 而不是 Cursor
        ↓
点击 [▾] 下拉按钮
        ↓
从菜单中选择 "Claude"
        ↓
按钮变为 [Paste to Claude]
        ↓
点击发送
```

## 五、产品边界

### 5.1 当前版本支持

- ✅ 区域截图
- ✅ 图片 + 文字粘贴
- ✅ 多应用支持（白名单）
- ✅ 本地保存

### 5.2 当前版本不支持

- ❌ 全屏/窗口截图模式
- ❌ 截图标注（箭头、文字、马赛克）
- ❌ 浏览器内 Web App 输入框自动聚焦
- ❌ 云端同步
- ❌ 历史记录

### 5.3 未来规划 (Roadmap)

| 阶段 | 功能 |
|-----|------|
| **v1.1** | 截图标注工具 |
| **v1.2** | 浏览器 Web App 自动聚焦 (ChatGPT, Gemini 等) |
| **v2.0** | 截图历史 + 云同步 |
