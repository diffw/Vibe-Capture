## Code Patterns

### Basic Test Structure
```swift
import XCTest

final class AccessibilityTests: XCTestCase {
    private let app = XCUIApplication()

    override func setUpWithError() throws {
        continueAfterFailure = false
        app.launch()
    }

    func testMainWindowLabelsExist() {
        XCTAssertFalse(app.buttons["capture-button"].label.isEmpty,
                       "Capture button must have an accessibility label")
    }

    func testKeyboardNavigationReachesAllActions() {
        app.typeKey(.tab, modifierFlags: [])
        let focused = app.focusedElement
        XCTAssertTrue(focused.exists, "Tab key must move focus to an element")
    }
}
```

## Platform-Specific Considerations

- macOS relies heavily on keyboard navigation — verify full Tab key traversal.
- Test VoiceOver labels: every interactive element must have a meaningful `.accessibilityLabel`.
- Verify focus ring visibility (system default; never override).
- Test with "Increase Contrast" enabled in System Settings for color contrast compliance.
- Toolbar items and menu bar extras also need accessibility labels.
- Verify right-click context menus are accessible via keyboard (Ctrl+click or menu key).
